    ): Promise<string | Buffer<ArrayBufferLike>>

  • Downloads an attachment from a message.

    This method retrieves an attachment from a specific message within a user's mailbox. The attachment can be returned as either binary data (Buffer) or a string, depending on the sendAsString parameter.

    Parameters

    • userId: string

      The user's ID.

    • mailboxId: string

      The ID of the mailbox.

    • messageId: number

      The ID of the message.

    • attachmentId: string

      The ID of the attachment.

    • OptionalsendAsString: boolean = false

      If true, the attachment is returned as a string. Otherwise, it is returned as a Buffer.

    Returns Promise<string | Buffer<ArrayBufferLike>>

    A promise that resolves to the attachment contents.

    Throws

    If the request fails.

    Example

    // Get attachment as binary data
    const attachmentBuffer = await messages.downloadAttachment(
      "user123",
      "mailbox456",
      789,
      "attachment001"
    );
    console.log(attachmentBuffer instanceof Buffer); // true

    Example

    // Get attachment as a string
    const attachmentString = await messages.downloadAttachment(
      "user123",
      "mailbox456",
      789,
      "attachment001",
      true
    );
    console.log(typeof attachmentString); // "string"

  • Search and update messages in a user's mailbox.

    This method applies specified actions (e.g., move, mark as seen, or flagged) to all matching messages. The operation is asynchronous, meaning it returns immediately while the actual modifications are processed in the background.

    Parameters

    • userId: string

      The user's ID.

    • params: {
          action: { flagged?: boolean; moveTo?: string; seen?: boolean };
          attachments?: boolean;
          dateend?: string;
          datestart?: string;
          flagged?: boolean;
          from?: string;
          id?: string;
          includeHeaders?: string;
          ip?: string;
          mailbox?: string;
          maxSize?: number;
          minSize?: number;
          q?: string;
          query?: string;
          searchable?: boolean;
          sess?: string;
          subject?: string;
          thread?: string;
          to?: string;
          unseen?: boolean;
      }

      Search criteria and actions to apply.

      • action: { flagged?: boolean; moveTo?: string; seen?: boolean }

        Actions to apply to the matching messages.

        • Optionalflagged?: boolean

          State of the \Flagged flag.

        • OptionalmoveTo?: string

          ID of the target mailbox to move messages to.

        • Optionalseen?: boolean

          State of the \Seen flag.

      • Optionalattachments?: boolean

        If true, matches only messages with attachments.

      • Optionaldateend?: string

        Datestring for the latest message storing time.

      • Optionaldatestart?: string

        Datestring for the earliest message storing time.

      • Optionalflagged?: boolean

        If true, matches only flagged messages.

      • Optionalfrom?: string

        Partial match for the "From" header.

      • Optionalid?: string

        Message ID values, used with mailbox.

      • OptionalincludeHeaders?: string

        Comma-separated list of headers to include in the response.

      • Optionalip?: string

        Optional IP address for logging.

      • Optionalmailbox?: string

        ID of the mailbox to search in.

      • OptionalmaxSize?: number

        Maximal message size in bytes.

      • OptionalminSize?: number

        Minimal message size in bytes.

      • Optionalq?: string

        Additional query string.

      • Optionalquery?: string

        Search string for full-text search.

      • Optionalsearchable?: boolean

        If true, excludes messages in Junk or Trash folders.

      • Optionalsess?: string

        Optional session identifier for logging.

      • Optionalsubject?: string

        Partial match for the "Subject" header.

      • Optionalthread?: string

        Thread ID.

      • Optionalto?: string

        Partial match for the "To" and "Cc" headers.

      • Optionalunseen?: boolean

        If true, matches only unseen messages.

    Returns Promise<{ existing: boolean; scheduled: string; success: boolean }>

    A promise that resolves to the operation status.
